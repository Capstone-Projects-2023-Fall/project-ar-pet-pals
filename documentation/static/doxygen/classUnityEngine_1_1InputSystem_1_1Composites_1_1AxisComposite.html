Detailed Description

A single axis value computed from one axis that pulls in the negative direction (minValue) and one axis that pulls in the positive direction (maxValue).

The limits of the axis are determined by minValue and maxValue. By default, they are set to [-1..1]. The values can be set as parameters.

var action = new InputAction();
action.AddCompositeBinding("Axis(minValue=0,maxValue=2")
.With("Negative", "<Keyboard>/a")
.With("Positive", "<Keyboard>/d");

If both axes are actuated at the same time, the behavior depends on whichSideWins. By default, neither side will win (WhichSideWins.Neither) and the result will be 0 (or, more precisely, the midpoint between minValue and maxValue). This can be customized to make the positive side win (WhichSideWins.Positive) or the negative one (WhichSideWins.Negative).

This is useful, for example, in a driving game where break should cancel out accelerate. By binding negative to the break control(s) and positive to the acceleration control(s), and setting whichSideWins to WhichSideWins.Negative, if the break button is pressed, it will always cause the acceleration button to be ignored.

The actual absolute values of negative and positive are used to scale minValue and maxValue respectively. So if, for example, positive is bound to Gamepad.rightTrigger and the trigger is at a value of 0.5, then the resulting value is maxValue * 0.5 (the actual formula is midPoint + (maxValue - midPoint) * positive).
