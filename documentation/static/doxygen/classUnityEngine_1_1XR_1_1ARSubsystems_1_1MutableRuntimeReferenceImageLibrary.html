Detailed Description

A reference image library that can be constructed and modified at runtime.

This differs from an XRReferenceImageLibrary, which can only be constructed in the Editor and is immutable at runtime.

‍[!IMPORTANT] Implementors: XRImageTrackingSubsystem providers must implement this class for their provider if XRImageTrackingSubsystemDescriptor.supportsMutableLibrary is true to provide the functionality to support runtime mutable libraries.

This is not something consumers of the UnityEngine.XR.ARSubsystems namespace should implement.

Asynchronously adds an image to this library.

Image addition can take some time (several frames) due to extra processing that must occur to insert the image into the library. This is done using the Unity Job System. The returned JobHandle can be used to chain together multiple tasks or to query for completion, but can be safely discarded if you do not need it.

This job, like all Unity jobs, can have dependencies (using the inputDeps ). This can be useful, for example, if imageBytes is the output of another job. If you are adding multiple images to the library, it is not necessary to pass a previous ScheduleAddImageJob JobHandle as the input dependency to the next ScheduleAddImageJob; they can be processed concurrently.

The imageBytes must be valid until this job completes. The caller is responsible for managing its memory.

Parameters
imageBytesThe raw image bytes in format .
sizeInPixelsThe width and height of the image, in pixels.
formatThe format of imageBytes . Test for and enumerate supported formats with supportedTextureFormatCount, GetSupportedTextureFormatAt(int), and IsTextureFormatSupported(TextureFormat).
referenceImageThe XRReferenceImage data associated with the image to add to the library. This includes information like physical dimensions, associated Texture2D (optional), and string name. The XRReferenceImage.guid must be set to zero (empty). A new guid is automatically generated for the new image.
inputDeps(Optional) input dependencies for the add image job.
Returns
A JobHandle which can be used to chain together multiple tasks or to query for completion. Can be safely discarded.
Exceptions
System.ArgumentExceptionThrown if imageBytes does not contain any bytes.
System.ArgumentExceptionThrown if referenceImage .guid is not Guid.Empty.
System.ArgumentNullExceptionThrown if referenceImage .name is null.
System.ArgumentOutOfRangeExceptionThrown if referenceImage .specifySize is true and referenceImage .size.x contains a value less than or equal to zero.
System.InvalidOperationExceptionThrown if the format is not supported. You can query for support using IsTextureFormatSupported.
System.ArgumentOutOfRangeExceptionThrown if sizeInPixels .x or sizeInPixels .y is less than or equal to zero.

Asynchronously adds an image to this library.

Image addition can take some time (for example, several frames) due to the processing that must occur to insert the image into the library. This is done using the [Unity Job System] (https://docs.unity3d.com/Manual/JobSystem.html). The returned AddReferenceImageJobState allows your to query for the status of the job, and, if the job completed, whether it was successful. The job state includes the JobHandle which can be used to chain together multiple tasks.

This job, like all Unity jobs, can have dependencies (using the inputDeps ). This can be useful, for example, if imageBytes is the output of another job. If you are adding multiple images to the library, it is not necessary to pass a previous ScheduleAddImageWithValidationJob's JobHandle as the input dependency to the next ScheduleAddImageWithValidationJob.

The imageBytes must be valid until this job completes. The caller is responsible for managing its memory. You can use the resulting JobHandle to schedule a job that deallocates the imageBytes .

Parameters
imageBytesThe raw image bytes in format .
sizeInPixelsThe width and height of the image, in pixels.
formatThe format of imageBytes . Test for and enumerate supported formats with supportedTextureFormatCount, GetSupportedTextureFormatAt(int), and IsTextureFormatSupported(TextureFormat).
referenceImageThe XRReferenceImage data associated with the image to add to the library. This includes information like physical dimensions, associated Texture2D (optional), and string name. The XRReferenceImage.guid must be set to zero (System.Guid.Empty). A new guid is automatically generated for the new image.
inputDeps(Optional) Input dependencies for the add image job.
Returns
Returns an AddReferenceImageJobState that can be used to query the status of the job. The AddReferenceImageJobState can be used to determine whether the image was successfully added. Invalid images will be not be added to the reference image library.
Exceptions
System.ArgumentExceptionThrown if imageBytes does not contain any bytes.
System.ArgumentExceptionThrown if referenceImage 's XRReferenceImage.guid is not System.Guid.Empty.
System.ArgumentNullExceptionThrown if referenceImage 's XRReferenceImage.name is null.
System.ArgumentOutOfRangeExceptionThrown if referenceImage 's XRReferenceImage.specifySize is true but referenceImage .XRReferenceImage.size contains values less than or equal to zero.
System.InvalidOperationExceptionThrown if the format is not supported. You can query for support using IsTextureFormatSupported.
System.ArgumentOutOfRangeExceptionThrown if sizeInPixels contains values less than or equal to zero.
