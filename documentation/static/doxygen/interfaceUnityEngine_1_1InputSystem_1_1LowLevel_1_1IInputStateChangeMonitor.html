Detailed Description

Interface used to monitor input state changes.

Use InputState.AddChangeMonitor(InputControl,IInputStateChangeMonitor,long,uint) to install a state change monitor receiving state change callbacks for a specific control.

See also

Called when the state monitored by a state change monitor has been modified.

REVIEW: For v2, consider changing the signature of this to put the "was consumed" signal outside the eventPtr

Parameters
controlControl that is being monitored by the state change monitor and that had its state memory changed.
timeTime on the InputEvent.time timeline at which the control state change was received.
eventPtrIf the state change was initiated by a state event (either a StateEvent or DeltaStateEvent), this is the pointer to that event. Otherwise it is pointer that is still InputEventPtr.valid, but refers a "dummy" event that is not a StateEvent or DeltaStateEvent.
monitorIndexIndex of the monitor as passed to InputState.AddChangeMonitor(InputControl,IInputStateChangeMonitor,long,uint).

To signal that the state change has been processed by the monitor and that no other pending notifications on the same monitor instance should be sent, set the InputEventPtr.handled flag to true on eventPtr . Note, however, that aside from only silencing change monitors on the same IInputStateChangeMonitor instance, it also only silences change monitors with the same groupIndex value as supplied to InputState.AddChangeMonitor(InputControl,IInputStateChangeMonitor,long,uint).
