Detailed Description

Information about an error condition including a domain, a domain-specific error code, and application-specific information.

This is a wrapper for Objective-C's NSError object.

Tests for equality.

This equality operator allows you to compare null with an NSError to determine whether its underlying pointer is null. This allows for a more natural comparison with the Objective-C object:

void TestForNull(NSError error)
{
// True if the native NSError object is null.
if (error == null)
{
// Do something with the error
}
}

lhsThe nullable NSError to compare with rhs .
rhsThe nullable NSError to compare with lhs .
Returns

Returns true if any of these conditions are met:

  • lhs and rhs are both not null and their underlying pointers are equal.
  • lhs is null and rhs 's underlying pointer is null.
  • rhs is null and lhs 's underlying pointer is null.
  • Both lhs and rhs are null.

Returns false otherwise.

The domain as an NSErrorDomain enum.

This method tries to determine the error domain without generating any managed strings. If the error domain is NSErrorDomain.Unknown, you might still be able to get additional information from its string representation (domain).
